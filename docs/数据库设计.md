# 工业互联网平台数据库设计

## 1. 数据库策略

项目默认使用 H2 文件数据库，数据源配置位于各后端服务的 `src/main/resources/application.yml`，例如 `platform-core-service` 与 `visual-video-service`：

```yaml
spring:
  datasource:
    url: jdbc:h2:file:./data/iot-platform-db;MODE=MySQL;DATABASE_TO_LOWER=TRUE;AUTO_SERVER=TRUE
    driver-class-name: org.h2.Driver
    username: sa
```

默认 H2 数据库优点：

- 启动简单，不依赖外部数据库。
- 适合项目验证和本地运行。
- JPA 可自动建表和更新表结构。
- 示例数据由 `DataInitializer` 在首次启动时初始化。

同时，项目提供 MySQL 8.x 交付脚本：

- `database/schema-mysql.sql`：建库建表脚本。
- `database/data-mysql.sql`：示例数据脚本。
- `database/README.md`：MySQL 切换说明。

后端 `pom.xml` 已包含 `mysql-connector-j` 运行时依赖。如需切换 MySQL，只需修改数据源 URL、用户名、密码和驱动。

## 2. 公共字段设计

多数业务实体继承 `BaseEntity`，包含以下公共字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | BIGINT | 主键，自增 |
| `name` | VARCHAR | 名称 |
| `code` | VARCHAR | 编码 |
| `status` | VARCHAR | 状态，如 ENABLED、DISABLED、ONLINE、RUNNING、OPEN、CLOSED |
| `remark` | VARCHAR | 备注 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

JPA 实体中通过 `@PrePersist` 和 `@PreUpdate` 自动维护创建时间和更新时间。

## 4. 主要表字段说明

### 4.1 用户与角色

#### `role`

| 字段 | 说明 |
| --- | --- |
| `name` | 角色名称，如超级管理员 |
| `code` | 角色编码，如 admin |
| `permissions` | 权限字符串，当前版本中使用 `*:*:*` 表示全部权限 |

#### `sys_user`

| 字段 | 说明 |
| --- | --- |
| `username` | 登录账号 |
| `password` | 登录密码，当前版本为简化本地账号模型；生产环境应使用哈希存储 |
| `real_name` | 真实姓名 |
| `role_name` | 角色名称 |

默认账号为 `admin / 123456`。

### 4.2 产品与设备

#### `product_category`

| 字段 | 说明 |
| --- | --- |
| `parent_id` | 父分类 ID，用于树形分类扩展 |

#### `product`

| 字段 | 说明 |
| --- | --- |
| `category_id` | 所属产品分类 ID |
| `protocol` | 协议类型，如 MQTT、HTTP |
| `manufacturer` | 厂商 |

#### `device_group`

| 字段 | 说明 |
| --- | --- |
| `parent_id` | 父分组 ID |

#### `device`

| 字段 | 说明 |
| --- | --- |
| `product_id` | 所属产品 ID |
| `group_id` | 所属设备分组 ID |
| `device_key` | 设备密钥/唯一标识 |
| `location` | 设备位置 |
| `latitude`、`longitude` | 经纬度 |
| `last_online_at` | 最后上线时间 |

### 4.3 遥测、规则和告警

#### `telemetry_data`

| 字段 | 说明 |
| --- | --- |

## 5. 数据关系说明

- 一个产品分类可以包含多个产品。
- 一个产品可以关联多个设备。
- 一个设备分组可以包含多个设备。
- 一个设备可以产生多条遥测数据。
- 一个设备可以触发多条告警。
- 一条规则可以产生多条规则审计记录。
- 一个定时任务可以产生多条任务日志。
- 一个视频设备可以关联多个拉流代理和视频告警任务。

## 6. 示例数据

系统首次启动时会初始化：

- 超级管理员角色和 `admin` 用户。
- 工业传感器、边缘网关等产品分类和产品。
- 一号车间、能源站等设备分组。
- 注塑机温度传感器、空压站边缘网关等设备。
- 温度超过 80℃ 自动告警规则。
- MQTT 和 HTTP 接入服务。
- 温湿度 JSON 解析脚本。
- 工厂运行总览大屏。
- 车间球机、拉流代理和安全帽识别任务。
- 设备在线状态巡检任务。
- 网关固件记录。
- 两条模拟遥测数据，其中高温数据会生成告警。

## 7. MySQL 切换说明

执行脚本：

```bash
mysql -uroot -p < database/schema-mysql.sql
mysql -uroot -p < database/data-mysql.sql
```

修改 `application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/industrial_iot_platform?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: your_password
    driver-class-name: com.mysql.cj.jdbc.Driver
```

同时建议将 JPA 方言调整为 MySQL 相关方言，或由 Spring Boot 自动识别。
