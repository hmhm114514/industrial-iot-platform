# 工业互联网平台需求分析报告

## 1. 项目背景

随着制造业数字化和工业互联网的发展，企业需要通过统一平台接入设备、采集数据、监控运行状态、处理告警并进行可视化展示。传统设备管理方式存在数据分散、告警滞后、运维效率低、业务系统割裂等问题，难以满足智能工厂和工业现场实时管理要求。

本课程项目参考 PandaX 物联网平台功能，建设一个面向课程实践验收的工业互联网平台原型。平台通过前后端分离架构实现设备管理、数据上报、规则告警、历史数据、任务管理、视频中心和系统设置等功能，重点完成“设备接入—数据上报—规则判断—告警处置—日志审计”的业务闭环。

## 2. 建设目标

项目目标包括：

1. 建设一个可本地运行的工业互联网平台原型。
2. 实现核心设备管理、遥测数据上报、规则告警和仪表板统计功能。
3. 覆盖 PandaX 功能说明中的主要模块，并通过演示化方式体现扩展能力。
4. 提供清晰的源代码结构、数据库脚本、运行说明和课程报告文档。
5. 支撑课程验收、答辩演示和后续扩展开发。

## 3. 用户角色分析

| 用户角色 | 主要职责 | 关注点 |
| --- | --- | --- |
| 平台管理员 | 管理用户、角色、设备、规则和系统配置 | 平台可用性、权限、安全和数据完整性 |
| 设备运维人员 | 维护设备台账、查看在线状态、处理设备异常 | 设备状态、数据上报、告警处置 |
| 生产管理人员 | 查看生产现场设备运行情况和统计报表 | 仪表板、趋势图、告警数量和运行效率 |
| 系统实施人员 | 部署平台、配置数据库、维护接口和脚本 | 部署简单、模块清晰、接口易扩展 |
| 课程验收人员 | 检查功能实现、运行结果和报告质量 | 功能覆盖、演示闭环、文档完整性 |

## 4. 业务范围

项目采用分层实现范围：

### 4.1 核心真实实现范围

- 登录认证与接口拦截。
- 产品分类、产品、设备分组和设备管理。
- 设备模拟遥测上报。
- 温度阈值规则判断。
- 自动生成告警和规则审计。
- 历史遥测数据查询。
- 历史告警查询和处置。
- 仪表板统计和图表展示。
- 任务启停与任务日志。
- 操作日志和登录日志。

### 4.2 扩展演示实现范围

- 网络通信服务管理。
- 解析脚本管理。
- 组态大屏管理和发布状态演示。
- 视频设备、流代理、视频广场和视频告警任务。
- 固件管理和升级状态。
- 服务监控页面。

### 4.3 文档说明范围

- AI 智能体。
- 开发工具和代码生成。
- 多租户、复杂权限、真实视频流、真实 MQTT 接入等后续扩展。

## 6. 非功能性需求

| 类别 | 需求 |
| --- | --- |
| 可运行性 | 项目应能在本地通过 Maven 和 npm 启动运行 |
| 易部署性 | 默认使用 H2 文件数据库，避免依赖外部数据库 |
| 可扩展性 | 后端采用控制器、服务、实体、仓库分层，前端采用模块化页面和 API 封装 |
| 可维护性 | 代码目录清晰，README 和文档完整 |
| 可演示性 | 核心链路可真实触发数据变化和告警变化 |
| 安全性 | 所有 `/api/**` 接口通过简化 Token 拦截保护，登录接口除外 |
| 兼容性 | 提供 MySQL 8.x 建表和示例数据脚本 |

## 7. 数据需求

平台需要维护以下核心数据：

- 用户和角色数据：账号、密码、真实姓名、角色、权限。
- 产品和设备数据：产品分类、产品协议、设备分组、设备 Key、设备位置和在线状态。
- 遥测数据：设备 ID、设备名称、温度、湿度、压力、原始 payload、上报时间。
- 规则数据：指标、运算符、阈值、启用状态、告警等级。
- 告警数据：设备、等级、类型、内容、状态、处理人、处理时间。
- 任务数据：任务名称、编码、Cron 表达式、运行状态、最近执行结果。
- 日志数据：规则审计、任务日志、操作日志、登录日志。

## 8. 接口需求

平台后端提供 REST API，统一前缀为 `/api`，前端通过 Axios 调用。典型接口包括：

- `POST /api/auth/login`：登录。
- `GET /api/dashboard/summary`：仪表板统计。
- `POST /api/telemetry/simulate`：模拟设备上报。
- `GET /api/telemetry`：历史遥测。
- `GET /api/alarms`：告警列表。
- `POST /api/alarms/{id}/handle`：处置告警。
- `/api/products`、`/api/devices`、`/api/rules`、`/api/tasks`：业务资源 CRUD。

## 9. 验收需求

验收时应满足：

1. 后端可通过 `mvn spring-boot:run` 或打包后运行。
2. 前端可通过 `npm install && npm run dev` 启动。
3. 使用 `admin / 123456` 可登录系统。
4. 可以查看仪表板统计和图表。
5. 可以查看设备列表并执行模拟上报。
6. 温度超过阈值时可以自动生成告警。
7. 关闭规则后，同样的高温上报不再生成告警。
8. 可以查看历史数据、历史告警并处置告警。
9. 可以查看操作日志、登录日志和任务日志。
10. 提供数据库脚本、README 和课程实践报告文档。
