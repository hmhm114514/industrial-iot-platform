## 2. 介绍准备

### 2.1 启动后端

```bash
cd backend && mvn -q -DskipTests package
mvn -pl platform-core-service spring-boot:run
mvn -pl visual-video-service spring-boot:run
mvn -pl gateway-service spring-boot:run
```

确认网关地址：`http://localhost:8080`；核心服务端口为 `8081`，可视化视频服务端口为 `8082`。前端统一通过网关访问 `/api/**`。

### 2.2 启动前端

```bash
cd frontend
npm install
npm run dev
```

确认前端地址：`http://localhost:5173`。

### 2.3 默认账号

```text
账号：admin
密码：123456
```

## 5. 技术说明问题与参考回答

### Q1：为什么选择 Spring Boot + Vue3？

答：Spring Boot 适合构建 REST API 和微服务工程，生态成熟，和 JPA、H2、MySQL 集成方便；Vue3 适合构建后台管理单页应用，配合 Element Plus 和 ECharts 可以快速完成表格、表单和图表展示。该组合便于按网关、核心业务、可视化视频等服务边界持续扩展。

### Q2：项目是否只是静态页面？

答：不是。项目核心闭环是真实实现的。设备模拟上报会调用后端接口，写入遥测数据，执行启用规则，生成规则审计和告警，告警可以被处置，仪表板统计也会变化。视频中心和组态大屏当前提供台账、配置、发布状态和通道状态等能力，并保留清晰的后续演进边界。

### Q3：为什么默认使用 H2，而不是直接使用 MySQL？

答：项目验证场景中，不同电脑的数据库环境可能不同。默认使用 H2 可以做到开箱即用，降低运行门槛。同时项目提供 MySQL 建表脚本、示例数据脚本和 MySQL 驱动依赖，便于后续切换到 MySQL。

### Q4：规则引擎是如何实现的？

答：规则表中保存指标、运算符、阈值、启用状态和告警等级。设备上报后，后端查询 `enabled=true` 的规则，根据上报指标进行比较，每次执行写入规则审计；如果命中规则，则生成告警记录。

### Q5：规则关闭后为什么不会继续触发告警？

答：规则启停接口会同步修改规则的 `status` 和 `enabled` 字段，后端执行规则时只查询 `enabled=true` 的规则。因此关闭规则后，该规则不会参与判断，也不会生成告警。

### Q6：系统的安全设计有哪些？

答：当前版本采用简化 Token 鉴权，登录成功后前端携带 `Authorization: Bearer panda-iot-demo-token` 访问接口，后端拦截 `/api/**` 请求。后续生产环境可以扩展为 JWT、RBAC 权限模型、密码加密和接口权限注解。

### Q7：视频中心和组态大屏的当前版本边界是什么？

答：当前版本提供视频设备台账、流代理配置、视频广场通道状态、视频告警任务、可视化大屏配置与发布。真实流媒体播放、GB28181 级联和拖拽组态编辑器属于后续演进计划，当前版本不声明为完整生产级实现。

### Q8：项目后续如何扩展？

答：可以从四个方向扩展：第一，接入真实 MQTT Broker 和设备 SDK；第二，升级 JWT 和 RBAC 权限；第三，增强规则引擎，支持多条件、联动动作和消息通知；第四，接入真实流媒体播放、GB28181 级联和拖拽组态编辑器。
