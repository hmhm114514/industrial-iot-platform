# 工业互联网平台开发实践周报

## 第一周：需求理解与选题确定

### 本周任务

- 阅读《工业互联网平台开发实践》实践指导书。
- 阅读 PandaX 物联网平台功能说明文档。
- 明确课程项目方向、开发范围和交付物。
- 初步确定系统技术栈和模块边界。

### 完成内容

- 确定选题为“工业互联网/物联网平台开发实践”。
- 梳理平台需要覆盖的主要能力：控制台、设备管理、服务管理、规则引擎、组态大屏、视频中心、数据中心、任务中心、日志系统和系统设置。
- 根据课程时间和验收要求，确定项目采用“完整可运行原型 + 核心闭环真实实现 + 扩展模块演示化实现”的建设方式。
- 确认核心演示链路为：管理员登录 → 设备管理 → 模拟设备上报 → 规则告警 → 历史数据/历史告警 → 告警处置 → 日志审计 → 仪表板统计更新。

### 遇到的问题与解决

| 问题 | 解决方式 |
| --- | --- |
| PandaX 功能点较多，全部深度实现成本较高 | 将功能划分为核心闭环、扩展演示和文档说明三类 |
| 课程要求既重视代码也重视报告 | 在项目结构中同时规划源码、数据库脚本和文档目录 |
| 部署环境不确定 | 默认采用 H2 文件数据库降低运行门槛，同时提供 MySQL 脚本 |

### 下周计划

- 完成项目总体规划和架构设计。
- 创建前后端工程骨架。
- 设计核心实体、接口和数据库表。

## 第二周：系统规划与后端开发

### 本周任务

- 设计前后端分离总体架构。
- 搭建 Spring Boot 后端工程。
- 实现登录、接口拦截、通用 CRUD 和核心业务服务。
- 初始化示例数据和数据库脚本。

### 完成内容

- 后端采用 Spring Boot 3 + Spring Web + Spring Data JPA。
- 默认数据库采用 H2，配置文件中提供 H2 Console，数据库文件位于 `backend/data/`。
- 实现简化 Token 鉴权，登录接口为 `POST /api/auth/login`。
- 实现产品分类、产品、设备分组、设备、规则、告警、任务、用户、角色、日志等实体和通用 CRUD 接口。
- 实现模拟遥测上报接口 `POST /api/telemetry/simulate`，可以写入遥测记录、更新设备在线状态、执行阈值规则并生成告警。
- 编写 MySQL 建表脚本和示例数据脚本。

### 遇到的问题与解决

| 问题 | 解决方式 |
| --- | --- |
| 多模块接口数量较多 | 抽象通用 CRUD 控制器，减少重复代码 |
| 规则引擎需要与遥测数据联动 | 在 `PlatformService.simulate` 中集中处理上报、规则审计和告警生成 |
| 需要兼顾 H2 与 MySQL | H2 用于默认运行，MySQL 脚本用于课程交付和扩展部署 |

### 下周计划

- 开发 Vue3 前端页面。
- 完成登录页、主布局、仪表板和主要业务模块页面。
- 与后端接口进行初步联调。

## 第三周：前端开发与页面集成

### 本周任务

- 搭建 Vue3 + Vite 前端工程。
- 实现登录页、主布局、导航菜单和仪表板。
- 实现设备管理、规则引擎、数据中心、任务中心等页面。
- 对接后端 REST API。

### 完成内容

- 前端采用 Vue3、Vite、Element Plus、ECharts 和 Axios。
- 实现登录页面，支持默认账号 `admin / 123456`。
- 主界面采用工业蓝视觉风格，包含侧边菜单、顶部栏、面包屑、主题切换和用户退出。
- 仪表板展示设备数量、在线设备、告警数量、今日消息、任务数量和趋势图。
- 通用资源页面支持查询、新增、编辑、删除、启停、导出等操作。
- 实现设备模拟上报、规则配置、历史数据、历史告警处置和任务启停等核心页面交互。
- 对组态大屏、视频广场、流媒体服务、固件管理和日志等模块进行演示化实现。

### 遇到的问题与解决

| 问题 | 解决方式 |
| --- | --- |
| 后端字段名和前端展示字段不完全一致 | 增加前端数据适配层 |
| 扩展模块后端功能较轻量 | 页面提供列表、表单、状态、预览等演示能力 |
| 图表库和组件库导致构建体积较大 | 作为课程演示非阻断问题记录，后续可通过路由懒加载优化 |

### 下周计划

- 进行前后端联调。
- 修复接口路径、字段映射和业务语义问题。
- 完成 README 和运行说明。

## 第四周：联调验证与问题修复

### 本周任务

- 对前后端接口进行联调。
- 验证核心演示链路。
- 修复规则启停、任务状态、服务监控等问题。
- 完成 README 和交付目录清理。

### 完成内容

- 修复前端 API 路径，使其与后端控制器保持一致。
- 修复设备模拟上报时 `deviceId` 未传递的问题。
- 修复规则保存时告警等级字段与后端模型不一致的问题。
- 修复规则启停语义，使关闭规则后高温上报不再触发告警。
- 修复任务保存状态，统一使用 `RUNNING/STOPPED` 与 `running` 字段。
- 增加 MySQL 驱动依赖并完善数据库说明。
- 清理 `node_modules`、`target`、`dist`、`data` 等构建和运行产物，保持交付目录整洁。

### 遇到的问题与解决

| 问题 | 解决方式 |
| --- | --- |
| 前端规则启停只改状态不改 `enabled` | 后端通用 toggle 对规则实体同步处理 `enabled` |
| 任务启停按钮可用但保存后状态显示不一致 | 前端保存时同时提交 `running` 和标准状态值 |
| MySQL 切换说明与依赖不完整 | 添加 `mysql-connector-j` 并完善说明文档 |

### 下周计划
