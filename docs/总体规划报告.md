# 工业互联网平台总体规划报告

## 1. 总体目标

本项目规划建设一个面向工业现场设备接入和运行监控的工业互联网平台原型。平台以设备管理和数据上报为基础，以规则告警和数据中心为核心，以仪表板、大屏、视频和日志审计为展示与运维支撑，形成可运行、可演示、可扩展的课程实践系统。

总体目标如下：

- 完成前后端分离平台工程。
- 完成设备、产品、规则、告警、任务、日志等核心数据模型。
- 完成“设备数据上报—规则判断—告警生成—告警处置—统计更新”闭环。
- 覆盖 PandaX 平台参考模块，体现工业互联网平台综合能力。
- 输出源码、数据库脚本、README 和课程文档。

## 2. 建设原则

| 原则 | 说明 |
| --- | --- |
| 实用优先 | 优先保证系统可以启动、演示和验收 |
| 核心闭环真实 | 关键业务链路必须真实写库和联动统计 |
| 模块覆盖完整 | 对 PandaX 参考功能进行模块化映射 |
| 部署简单 | 默认使用 H2 文件数据库，降低环境依赖 |
| 易于扩展 | 预留 MySQL、真实设备接入、权限升级和消息中间件扩展空间 |
| 文档同步 | 设计、实现、测试和验收文档与源码保持一致 |

## 3. 技术路线

### 3.1 后端技术路线

- Java 17 目标版本。
- Spring Boot 3 作为后端基础框架。
- Spring Web 提供 REST API。
- Spring Data JPA 负责数据访问。
- H2 文件数据库作为默认运行数据库。
- MySQL 8.x 脚本用于课程交付和扩展部署。
- 简化 Token 鉴权保护 `/api/**` 接口。

### 3.2 前端技术路线

- Vue3 作为单页应用框架。
- Vite 作为开发和构建工具。
- Element Plus 提供后台管理界面组件。
- ECharts 提供统计图表。
- Axios 封装后端请求。
- Vite Proxy 将 `/api` 转发到后端 `http://localhost:8080`。

### 3.3 数据库技术路线

- 本地演示优先使用 H2，启动时由 JPA 自动建表并初始化示例数据。
- MySQL 使用 `schema-mysql.sql` 建表，`data-mysql.sql` 初始化示例数据。
- H2 采用 MySQL 兼容模式，降低迁移差异。

## 4. 模块规划

| 模块 | 子功能 | 实现方式 |
| --- | --- | --- |
| 控制台 | 统计卡片、图表、服务监控 | 后端统计接口 + 前端 ECharts |
| 设备管理 | 产品分类、产品、设备分组、设备、模拟上报 | 真实 CRUD + 模拟上报接口 |
| 服务管理 | 网络服务、解析脚本 | CRUD + 状态启停演示 |
| 规则引擎 | 阈值规则、规则启停、规则审计 | 真实规则执行和审计 |
| 数据中心 | 历史遥测、历史告警、告警处置 | 真实查询和处置 |
| 组态大屏 | 大屏分组、发布、预览 | 演示化配置和发布状态 |
| 视频中心 | 视频设备、拉流代理、视频广场、视频告警 | 演示化台账和页面 |
| 任务中心 | 定时任务、任务启停、任务日志 | 真实状态切换和日志生成 |
| 系统设置 | 用户、角色、固件、操作日志、登录日志 | CRUD + 审计展示 |

## 7. 风险控制

| 风险 | 影响 | 控制措施 |
| --- | --- | --- |
| 功能范围过大 | 开发周期不足 | 分为核心真实实现、扩展演示和文档说明三层 |
| 环境依赖复杂 | 老师验收时难以运行 | 默认使用 H2，避免强依赖 MySQL |
| 前后端接口不一致 | 页面无法正常展示 | 统一封装 API，联调后修正路径和字段适配 |
| 规则语义错误 | 核心闭环不可信 | 验证规则启用和关闭两种场景 |
| 文档与代码不一致 | 影响课程报告质量 | 以实际代码、README 和验证结果为依据编写文档 |
| 提交目录过大 | 压缩包或提交不规范 | 清理 `node_modules`、`target`、`dist` 和运行数据库文件 |
